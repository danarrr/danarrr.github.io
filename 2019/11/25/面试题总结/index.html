基础题

1. 简单描述下闭包。以及闭包的作用域？如果使用很多个闭包有什么坏处？

闭包的理解： 函数 A 内有函数 B，函数 B 可以访问到函数 A 的变量.

从上面的扩展例子会看到闭包内的值会一直保存在内存当中，
缺点： 从而引发得出如果编写多个闭包会使得内存溢出，因为不会在调用结束后，被垃圾回收机制 gc（garbage collection）回收。内存消耗很大。
特点： 1.缓存 2.面向对象中的对象 3.实现封装，防止变量跑到外层作用域中，发生命名冲突，可以保护变量作用域 4.匿名自执行函数，匿名自执行函数可以减小内存消耗

2. 什么是原型和原型链？举例说下编译前编译后，组件间 constructor 和 this 的指向。

  • prototype 和 proto
    每个对象都有一个proto属性，并且指向它的 prototype 原型对象
    每个构造函数都有一个 prototype 原型对象,prototype 原型对象里的 constructor指向构造函数本身

  • 原型链
    每个对象都有一个proto，它指向它的 prototype 原型对象，而 prototype 原型对象又具有一个自己的 prototype 原型对象，就这样层层往上直到一个对象的原型 prototype 为 null
    这个查询的路径就是原型链

  • JavaScript 中的继承
    属性继承

3. 简述下重绘和重排？如何优化？

重排：我们对 DOM 的修改引发了 DOM 几何元素的变化，渲染树（渲染树第四题有介绍）需要重新计算,性能消耗大.

  • 场景：
    1. 页面第一次渲染
    2. 窗口尺寸大小改变
    3. 新增删除可见元素
    4. 查询某些属性例如 offsetTop, clientTop…等计算窗口大小属性。

重绘： 样式变化，使浏览器需要根据新的属性进行绘制。

  • 衍生出来的面试题：
  1. display：none; ovibility: hidden; opacity: 0;三者间有什么区别？
    答：除了 dom 节点是否继续存在之外，display：none 会引起重排，消耗性能。
  2. 如何减少重排？
    答:
    方案一： 需要计算大小和变化大小的元素或者动画尽量脱离文档流, 例如修改定位：则不会引起重排.
    方案二：启用 GPU 加速： GPU 硬件加速是指应用 GPU 的图形性能对浏览器中的一些图形操作交给 GPU 来完成，因为 GPU 是专门为处理图形而设计，所以它在速度和能耗上更有效率。
    GPU 加速通常包括以下几个部分：Canvas2D，布局合成, CSS3 转换，CSS3 3D 变换，WebGL 和视频。

4. 浏览器渲染页面过程。

前提： 输入网址 -> dns 解析（如果搜索的域名找不到对应的内网 ip, 则开始解析） -> TCP 三次握手，确认可以正常通信 -> 客户端接收服务器发送过来的数据
开始渲染页面：

  1. 浏览器解析 html 源码，然后创建一个 dom 树，并行请求 js/css/image 在 dom 树中。dom 树根节点就是 documentElement.
    |
  2. 浏览器解析 css 代码，构建 CSSOM 树。解析 css 树优先级如下：
    html 中的 style > 内联样式 > 大于外联样式 > 用户设置 > 大于浏览器默认
    |
  3. 渲染树 render Tree = cssOM 树 + dom 树
    渲染树会忽略掉不需要渲染的元素，比如 head、display:none 的元素等。
    |
  4. 一旦渲染树创建好了，浏览器就可以根据渲染树直接把页面绘制到屏幕上。

以上四个步骤并不是一次性顺序完成的。如果 DOM 或者 CSSOM 被修改，以上过程会被重复执行。实际上，CSS 和 JavaScript 往往会多次修改 DOM 或者 CSSOM。
