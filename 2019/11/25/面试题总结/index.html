面试题总结

发表于 2019-11-25 | 分类于 javascript | | 阅读次数:

1. 简单描述下闭包。以及闭包的作用域？如果使用很多个闭包有什么坏处？

闭包的理解： 函数A内有函数B，函数B可以访问到函数A的变量.

从上面的扩展例子会看到闭包内的值会一直保存在内存当中，
缺点： 从而引发得出如果编写多个闭包会使得内存溢出，因为不会在调用结束后，被垃圾回收机制gc（garbage collection）回收。内存消耗很大。
特点：
1.缓存
2.面向对象中的对象
3.实现封装，防止变量跑到外层作用域中，发生命名冲突，可以保护变量作用域
4.匿名自执行函数，匿名自执行函数可以减小内存消耗

3. 简述下重绘和重排？如何优化？

重排：我们对DOM的修改引发了DOM几何元素的变化，渲染树（渲染树第四题有介绍）需要重新计算,性能消耗大.

  • 场景：

    1. 页面第一次渲染
    2. 窗口尺寸大小改变
    3. 新增删除可见元素
    4. 查询某些属性例如offsetTop, clientTop…等计算窗口大小属性。
      重绘： 样式变化，使浏览器需要根据新的属性进行绘制。

  • 衍生出来的面试题：

  1. display：none; ovibility: hidden; opacity: 0;三者间有什么区别？
    答：除了dom节点是否继续存在之外，display：none会引起重排，消耗性能。
  2. 如何减少重排？
    答:
    方案一： 需要计算大小和变化大小的元素或者动画尽量脱离文档流, 例如修改定位：则不会引起重排.
    方案二：启用GPU加速： GPU 硬件加速是指应用 GPU 的图形性能对浏览器中的一些图形操作交给 GPU 来完成，因为 GPU 是专门为处理图形而设计，所以它在速度和能耗上更有效率。

GPU 加速通常包括以下几个部分：Canvas2D，布局合成, CSS3转换（s），CSS3 3D变换（），WebGL和视频()。

4. 浏览器渲染页面过程。

前提： 输入网址 -> dns解析（如果搜索的域名找不到对应的内网ip, 则开始解析） -> TCP三次握手，确认可以正常通信 -> 客户端接收服务器发送过来的数据
开始渲染页面：

  1. 浏览器解析html源码，然后创建一个dom树，并行请求js/css/image 在dom树中。dom树根节点就是documentElement.
    |
  2. 浏览器解析css代码，构建CSSOM树。解析css树优先级如下：
    html中的style > 内联样式 > 大于外联样式 > 用户设置 > 大于浏览器默认
    |
  3. 渲染树render Tree = cssOM树 + dom树
    渲染树会忽略掉不需要渲染的元素，比如head、display:none的元素等。
    |
  4. 一旦渲染树创建好了，浏览器就可以根据渲染树直接把页面绘制到屏幕上。

以上四个步骤并不是一次性顺序完成的。如果DOM或者CSSOM被修改，以上过程会被重复执行。实际上，CSS和JavaScript往往会多次修改DOM或者CSSOM。
