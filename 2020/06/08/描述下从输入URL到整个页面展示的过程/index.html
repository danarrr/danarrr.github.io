描述下从输入URL到整个页面展示的过程

发表于 2020-06-08 | 阅读次数:

感觉这个论点我是看一次忘一次，这次整理了图形完整撸一遍流程~ （多文字慎入）

整个完成的流程

重点

  1. dns 解析
  2. tcp 三次握手 客户端-服务器-客户端
  3. http 请求与相应报文
  4. 拿到服务器相应的内容后，浏览器开始渲染内容（详细拆分点）
  5. Chrome v8 引擎通过代码编码过后转成的AST树， CPU开始绘制图形
dns 解析与预解析

我们在发布网站的时候，最终有一个步骤就是将服务器与域名绑定。所以当访问一个页面时候第一步就是解析域名，找到对应的 ip address。
DNS 是一个网络服务器，域名解析简单来说就是在 dns 上记录一条信息。

当我们访问页面时, 如果访问的域名在本地已经有映射 host，例如：

则会进行预解析，如果没有则直接进行 dns 解析。

那是如何通过域名去找到对应的 ip 的：
优先顺序） 从下面缓存中优先查询： 浏览器缓存 -> 操作系统缓存 -> 路由缓存 -> ISP(运营商)的服务器 -> 根域服务器，然后一层一层递归查询(DNS 服务器先问根域名服务器.com 域名服务器的 IP 地址，然后再问.baidu 域名服务器，依次类推)。

TCP 三次握手

顺序： 客户端 - 服务器 - 客户端

发送 HTTP 请求

客户端请求的过程中，请求行包含了请求头 url 还有协议版本号
请求头包含了： 很多附加的信息，基本上是键值对组成的，还有很多和客户端有关的信息，还有请求的信息。
请求体包含了： 请求参数和符号, 比如 get 请求带上的请求参数

服务器处理请求并返回报文
服务器： 网络中高性能的计算机

http 响应报文 包括状态码 协议版本 状态描述

拿到服务器返回的数据后，开始渲染页面

其实这部分才是前端接触频率最高的部分：

1.解析 dom 树

根据返回的 html 文件信息解析 dom 树。遍历解析 dom 然后构建节点。

  • 浏览器解码 将接收到的字节流转成unicode码
  • 根据语义创建对应的节点
  • 将节点关联到一起然后创建完整的 dom 树。
2.根据 css 规则解析 css 树，根据 css 优先级优先渲染
3. 根据 js 文件，一般是为了动态解析 dom 树
4. 根据渲染树和渲染内容，计算出每一个渲染对象的位置还有尺寸，开始布局
5. 布局完成之后就开始绘制页面

绘制阶段：系统会遍历呈现数
重绘：具体某个元素的字体背景色 不会影响其他 div 的布局
回流： 某个元素发生了尺寸或者位置的变化，则重新需要计算渲染树，重新渲染

数据传输完毕再进行 tcp 四次挥手,断开 tcp 连接。
  • 为什么要四次挥手
GPU 图形处理器 来绘制界面

补充

计算机是如何识别你写的这一行代码的？
计算机编码，之所以叫编码其实就是将我们编写的代码(某种语言)通过某个中介(编码过程)，编译成二进制，然后浏览器引擎去编译语言。

词法分析 和语法分析：语法分析就是讲词法分析转化为一个有层级的 抽象语法树 AST
babel 转码：就是先将 ES6 的代码编成 AST，然后转换成 ES5 的 AST
