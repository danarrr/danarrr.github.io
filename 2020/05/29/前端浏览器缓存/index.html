前端浏览器缓存

发表于 2020-05-29 | 阅读次数:

HTTP 头部信息控制缓存
大致分为两种： 强缓存和协商缓存
强缓存： 如果命中缓存，则不需要继续和服务器进行通信
协商缓存： 不管命不命中 都要进行通信

强制缓存： 可以理解为无需验证的强缓存策略，对强缓存来说，在响应头中看到强缓存的标识

  • expires
    expires
  • Cache-Control
    会给配置值 max-age 代表最后的相对时间，举例设置有效期是 3600 秒 那么就是有效时长
    可以强制设置不缓存 no-store

协商缓存
如果没有命中强制缓存，浏览器会发送请求到服务器, 请求会响应头新型

  • last-modified:记录最后修改的时间，启用强制缓存， 当再次请求资源的时候请求头会带上 if-modified-since 字段, 如果和之前返回的 last-modified 的值相同，则会告诉浏览器直接使用缓存并返回 304 标识符是最后更新时间

  • etag 的标识符是根据资源内容生成的一窜编码 缺点 生成标识符会增加服务器开销 优点 更加精准

机制、 整一个流程
强缓存优先 协商缓存 、 服务器重新获取资源
