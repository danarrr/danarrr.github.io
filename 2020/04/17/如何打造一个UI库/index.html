如何打造一个UI库

发表于 2020-04-17 | 阅读次数:

说在前面

工程化现在基本上是搭建项目的主流了，这里开发的 UI 库参考了 Antdesign的代码设计架构。有优秀成熟的方案不用花费过多的时间去摸索一整套构建方式。

通过调研项目参考源码 -> 参考项目结构如何工程化拆分, 构建方案 -> 前期投入较多时间成本但是后期减少很多痛点。

设计项目

先来看下参考 AntDesign 官网项目结构拆分:
├── config （ 配置文件 ）
│ ├── entry.js (运行命令，自动化生成页面）
│ └── readme.js （ 运行命令，编译/components/{md} 生成 html 文件）
├── package （npm 包）
│ ├── components
│ ├── index.ts
│ ├── style
│ ├── svgs （ icons 库图标 ）
│ └── utils（ 公共文件 ）
├── src （ 潭州课堂-UI 库官网界面）
│ ├── main.ts
│ └── views （ 页面 ）

主要拆分成核心的三个部分:

  1. 官网界面 ./src
  • 包括了组件的介绍文档
  • 组件的使用示例
  1. 组件 npm 包 ./package
  • 基础组件
  • 全局组件/指令，例如 Message
  • svg 文件导出
  1. 自动化配置文件 ./config
  • 编译组件介绍文档(markdown 文件), 并插入官网界面。
  • 自动化新增组件：包括新增 npm 包，官网介绍页，增加对应路由。

打包编译方式

打包工具

最终我选用了rollup打包编译组件库，调研以及实践结果如下：

1. rollup / gulp: rollup 对vue的打包更友好，生态圈完善，因为我使用了vue-prototype这个插件编写ts,gulp无法编译。
2. react编写ui库的话建议还是用gulp打包。
3. webpack / rollup: webpack更适用于打包业务项目Library，webpack在分模块打包这块做的更精致。rollup是适用于打包开源组件库，rollup主打打出更轻量级的包，主要功臣还是tree-shaking（尽管webpack2也已经用上tree-shaking)。

处理 svg

实现 svg sprite, 首先要了解 svg 的两种基本使用方式。

  1. 在 html 中插入具体的 svg 文件。
  2. 在 html 文档前就先引入所有 svg 文件，在使用时直接加上对应 id 名即可。

svgSprite 实现思路如下：

  1. 配置 rollup，使用 rollup-plugin-svg-icons 插件导入所有的 svg 文件。webpack 可以直接使用 requireAll Api 导入所有的 svg 文件。
  2. 把生成的 svg 总文件插入文档流。
    这样就可以完美支持自己写的组件了，使用时只需要添加 svg 文件 id 名就可以使用了。
