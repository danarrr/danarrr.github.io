初次接触babel7

发表于 2020-02-20 | 分类于 编译打包 | 阅读次数:

题外话

最近在开发脚手架过程中接入了 babel7 来编译故有了下文，编译的出发点需要多考虑如何让用户无感接入脚手架，
例如：用户没有配置ESlint或TSconfig的情况下自动生成一份配置文件并自动编译，等脚手架开发完再回来展述吧。。。

运行方式

众所周知 babel 可用于编译 js, ES678 to ES5, 兼容一些浏览器的运行。
babel 总共分为三个阶段：解析，转换，生成。

preset and plugin

preset 配置其实是整合了一部分插件，让用户在需要快速构建项目的时候不用花费过多的时间去自定义安装配置插件。

先说一下他们的执行顺序

  • Plugin 会运行在 Preset 之前。
  • Plugin 会从前到后顺序执行。
  • Preset 的顺序则 刚好相反(从后向前)。

编译插件

env

env最为常用, 官方介绍是取代了以往所有版本的 preset，转换新的 JavaScript 语法。
babel-preset-es2015、babel-preset-es2016、babel-preset-es2017、babel-preset-latest

配置@babel/preset-env需要指定执行环境的Browserlist, 可以参考如下配置。

polyfill

babel 编译器只会用来转换 js, 不会转换新的 API，@babel/polyfill可以转义所有新的 API，但是存在缺点：

会污染全局变量，给很多类的原型链上都作了修改，如果我们开发的也是一个类库供其他开发者使用，这种情况就会变得非常不可控。并且打包出来会非常大。

runtime

所以推荐配置 @babel/plugin-transform-runtime, 需要依赖引入@babel/runtime。
但是没有转换一些内置的类还有静态方法。通过引入corejs转换一些内置类 (Promise, Symbols 等等) 和静态方法 (Array.from 等)。
